## 문제 원인

파일 업로드 요청은 정상적으로 발급되지만, 실제 저장소 업로드 단계에서 `Invalid key` 오류가 발생하고 있습니다. 현재 파일 경로가 `1|1000|402-B/...`처럼 세션 ID의 `|` 문자를 그대로 포함해 생성되어 저장소 객체 키 규칙에 걸리는 것이 원인입니다. 그래서 버튼이 `업로드 중…`으로 바뀐 뒤 실패하고 다시 `확인`으로 돌아옵니다.

## 수정 계획

1. **업로드 파일 경로 안전화**
   - `requestBookmarkUpload`에서 저장소 경로에 사용할 세션 폴더명을 안전한 형태로 인코딩합니다.
   - 예: `1|1000|402-B` → URL-safe/base64url 형태의 폴더명.
   - 파일명은 기존처럼 한글 파일명을 보존하되 위험 문자 제거는 유지합니다.

2. **파일 경로 검증 로직 동기화**
   - `createBookmark`, `deleteBookmarkUpload`에서 기존 `filePath.startsWith(sessionId + "/")` 검증을 새 안전 폴더명 기준으로 바꿉니다.
   - 업로드된 파일 존재 확인 시 폴더 추출도 새 경로 구조와 맞도록 유지합니다.

3. **실패 메시지 명확화**
   - 브라우저 업로드 실패 시 `Invalid key` 같은 내부 오류 대신 “파일 업로드 경로를 만들 수 없습니다/업로드에 실패했습니다”처럼 사용자가 이해할 수 있는 안내가 뜨도록 보완합니다.

4. **영향 범위 제한**
   - 데이터베이스 구조, 권한, 저장소 정책은 변경하지 않습니다.
   - 기존 바로가기 링크 등록, 목록 조회, 삭제 기능은 그대로 유지합니다.
   - 이미 등록된 항목이 있다면 기존 파일 경로도 조회/삭제가 가능하도록 새 검증은 세션별 안전 경로만 엄격히 적용하되 필요한 범위에서 호환성을 고려합니다.

## 확인 방법

- 같은 PDF 파일을 선택하고 `확인` 클릭 시 모달이 닫히고 바로가기 칩이 추가되는지 확인합니다.
- 네트워크에서 저장소 업로드가 400이 아닌 성공으로 끝나는지 확인합니다.
- 추가된 파일 바로가기가 청중 화면에서도 정상 표시/다운로드되는지 확인합니다.
