## 목표
파일을 첨부한 바로가기 추가 시 한글 파일명이 저장소 객체 키에 들어가 400 오류로 실패하는 문제를 고칩니다. 링크만 있는 바로가기 생성 흐름은 유지합니다.

## 수정 계획
1. **업로드 방식 수정**
   - 저장소 객체 경로에는 `uuid + 확장자`만 사용해 한글/공백/특수문자가 객체 키에 들어가지 않게 합니다.
   - 사용자에게 보이는 원본 파일명은 DB의 `fileName`으로 그대로 저장해 다운로드 파일명에 사용합니다.

2. **오류 메시지 개선**
   - 업로드 400/실패 응답의 본문을 읽어 사용자에게 “파일 업로드에 실패했습니다” 형태로 보여주고, 내부 원인도 개발 중 확인 가능하게 정리합니다.
   - 실패 시 기존처럼 업로드된 orphan 파일 정리 흐름은 유지합니다.

3. **기존 흐름 유지**
   - 링크만 등록하는 경우, 링크+파일을 함께 등록하는 경우, 파일만 등록하는 경우 모두 현재 UI와 데이터 구조를 유지합니다.
   - DB/권한/스토리지 정책은 변경하지 않습니다.

4. **검증**
   - 서버 함수의 서명 URL 발급 → 브라우저 업로드 → `createBookmark` 호출 순서가 정상인지 확인합니다.
   - 링크만 등록과 파일 첨부 등록이 모두 성공하는지 네트워크/화면 동작 기준으로 확인합니다.
