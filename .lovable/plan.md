## 목표
PC 화면에서 질문이 많아져도 오른쪽 `질문 목록` 카드 자체가 계속 길어지지 않고, 카드 내부 목록 영역에만 세로 스크롤이 생기도록 고정합니다.

## 수정 계획
1. `src/routes/presenter.tsx`의 PC 전용 레이아웃 높이 기준을 명확히 고정합니다.
   - `ResizablePanelGroup`의 `min-h-[600px]`가 화면 높이보다 커지며 전체 컨테이너가 늘어나는 원인이 될 수 있으므로, PC에서는 실제 뷰포트 기준 높이를 넘지 않도록 조정합니다.

2. 오른쪽 컬럼과 질문 목록 카드에 `min-h-0`, `h-full`, `overflow-hidden`을 더 강하게 적용합니다.
   - 부모 flex/grid 컨테이너가 자식 콘텐츠 높이만큼 커지는 현상을 막습니다.

3. 질문 목록 스크롤 영역을 명시적인 내부 스크롤 컨테이너로 고정합니다.
   - 현재 `h-0 flex-1 overflow-y-auto` 구조를 유지하되, 필요하면 `min-h-0`과 `pr-1`을 추가해 카드 내부에서만 스크롤되도록 만듭니다.

4. 모바일/태블릿의 기존 동작은 유지합니다.
   - `xl:hidden` 이하 레이아웃과 `max-h-[70vh]` 제한은 건드리지 않습니다.

## 예상 영향
- 기능 로직, 데이터, 서버 호출은 변경하지 않습니다.
- 서버비 증가 가능성은 없습니다.
- 변경 범위는 발표자 화면의 레이아웃 CSS에만 제한됩니다.
- 질문 카드 버튼/정렬/상태 필터 기능에는 영향이 없습니다.

## 검증 기준
- PC 화면에서 질문 6개 이상일 때 오른쪽 `질문 목록` 카드 높이가 화면 안에 고정됩니다.
- 질문 리스트만 카드 내부에서 세로 스크롤됩니다.
- 왼쪽 토핑 영역과 상단 제어 영역은 기존처럼 표시됩니다.
