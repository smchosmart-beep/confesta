## 목표
청중(핑크) 트럭의 옆면 유리창 3개(좌측 운전석 창 + 중앙 작은 창 + 우측 창) 형태를 그대로 가져와, 나머지 3대 트럭(운영 스태프-민트, 관리자-망고, 발표자-블루)에 완전 교체 적용합니다. 기존 유리창 잔상은 남기지 않습니다.

## 작업 방식
관리자 트럭 차양 교체 때와 동일한 픽셀 레벨 교체 흐름:

1. **청중 트럭 유리창 추출**
   - `truck-audience.png` 자산을 받아 좌/중/우 유리창 영역의 마스크와 모양을 PIL로 정확히 추출
   - 유리창 외곽선·내부 하이라이트·반사 디테일을 알파 채널과 함께 보존

2. **각 대상 트럭에서 기존 유리창 완전 제거**
   - 운영 스태프(민트), 관리자(망고), 발표자(블루) 트럭 이미지에서 기존 창문 영역(잔상 포함)을 차체 색으로 인페인팅 → 완전 제거
   - 잔상이 남지 않는지 확대 검증

3. **청중 트럭 유리창 합성**
   - 추출한 3개 창문 모양을 각 트럭의 동일 위치에 그대로 합성
   - 유리 내부 색조만 각 트럭의 톤(민트/망고/블루)에 살짝 맞춰 자연스럽게(투명도 + 차체색 반영), 형태·테두리·하이라이트는 100% 청중 트럭과 동일
   - 텍스트/아이콘/아이스크림콘/차양/바퀴 등 다른 요소는 절대 건드리지 않음

4. **검증**
   - 모바일 뷰포트에서 확대 스크린샷 → 3개 트럭 모두 청중 트럭과 창문 형태가 동일한지, 기존 창문 잔상이 없는지 확인
   - 문제 있으면 재합성

5. **자산 교체**
   - `lovable-assets create`로 세 트럭 이미지 업로드 후 각 `.asset.json` 갱신
   - 코드 변경 없음, 자산만 교체

## 적용 대상 자산

- `src/assets/truck-mango.png.asset.json` (망고)

기준 자산: `src/assets/truck-audience.png.asset.json` (핑크)

## 하지 않는 것
- AI 이미지 재생성 사용 안 함 (관리자 트럭 차양은 마지막에 AI 썼지만, 이번엔 먼저 픽셀 교체로 시도해 잔상 없이 끝내는 것을 1순위)
- 컴포넌트/레이아웃/텍스트 변경 없음
