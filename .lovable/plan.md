## 목표
파일을 첨부한 바로가기 추가 시 400 오류로 실패하는 문제를 고칩니다. 링크만 있는 바로가기 생성 흐름은 유지합니다.

## 수정 계획
1. **업로드 방식 수정**
   - 현재 브라우저에서 `uploadToSignedUrl(req.filePath, req.token, file, ...)`를 호출하는데, 서명 업로드 API는 `path`가 아니라 `token` 중심으로 처리되어 400이 날 가능성이 큽니다.
   - 반환받은 `uploadUrl`을 직접 `fetch(..., { method: 'PUT', body: file })`로 호출하는 방식으로 바꿔, 서버가 발급한 서명 URL 그대로 업로드되게 합니다.

2. **오류 메시지 개선**
   - 업로드 400/실패 응답의 본문을 읽어 사용자에게 “파일 업로드에 실패했습니다” 형태로 보여주고, 내부 원인도 개발 중 확인 가능하게 정리합니다.
   - 실패 시 기존처럼 업로드된 orphan 파일 정리 흐름은 유지합니다.

3. **기존 흐름 유지**
   - 링크만 등록하는 경우, 링크+파일을 함께 등록하는 경우, 파일만 등록하는 경우 모두 현재 UI와 데이터 구조를 유지합니다.
   - DB/권한/스토리지 정책은 변경하지 않습니다.

4. **검증**
   - 서버 함수의 서명 URL 발급 → 브라우저 업로드 → `createBookmark` 호출 순서가 정상인지 확인합니다.
   - 링크만 등록과 파일 첨부 등록이 모두 성공하는지 네트워크/화면 동작 기준으로 확인합니다.
