
## 문제

현재 4개 트럭 이미지는 각기 독립적으로 AI 생성되어 (1) 차체 모양·각도·크기가 다르고 (2) 지붕의 아이스크림 콘이 비율·각도가 어색합니다 (스태프 트럭은 콘이 글자에 닿을 만큼 큼, 발표자 트럭은 콘이 잘림).

## 해결 방안

**1개의 "기준 트럭"을 먼저 만들고, 색만 바꿔 4개 변형 생성** — 이렇게 해야 모양·크기·아이스크림 위치가 완전히 동일합니다.

### 단계

1. **기준 트럭 1장 생성** (`/tmp/truck-base.png`)
   - 정면 측면 뷰, 정사각형 1024×1024, 투명 배경
   - 프롬프트에 명시: 아이스크림 콘은 트럭 지붕 중앙에 작고 안정적으로 (트럭 높이의 약 25%), 콘이 기울지 않고 똑바로, 스쿱 1개만, 가장자리 잘림 없음
   - 색상은 일단 화이트/뉴트럴 기조 (이후 리컬러용 베이스)

2. **`imagegen edit_image`로 4색 변형 생성**
   - 동일한 베이스 이미지를 입력으로 4번 호출
   - 프롬프트: "차체 색상만 X로 변경, 모양·구도·크기·아이스크림 콘 위치 모두 그대로 유지"
   - 4색: 딸기 핑크 / 블루베리 라벤더블루 / 민트 그린 / 망고 오렌지
   - 각각 아이스크림 스쿱 색만 차체와 어울리게 함께 변경

3. **에셋 교체**
   - 기존 4개 `src/assets/truck-*.png.asset.json` 를 `delete_asset` 으로 제거
   - 새 4장을 `lovable-assets create` 로 업로드, 같은 파일명으로 재생성
   - `TruckCard.tsx` 는 코드 변경 불필요 (import 경로 동일)

4. **검증**
   - Playwright로 메인화면 스크린샷 → 4개 카드가 동일한 트럭 실루엣을 갖는지, 콘이 카드 위쪽에서 잘리지 않는지 확인
   - 필요시 `TruckCard.tsx`의 이미지 컨테이너 `aspect-ratio` 또는 `object-position`만 미세조정

### 리스크 / 대안

- `edit_image` 리컬러가 실패하거나 모양이 변형되면 → 대안: 단일 베이스 PNG를 그대로 4번 import하고 CSS `filter: hue-rotate()` 로 색만 변경 (모양 100% 동일 보장, 단 색조 정밀도 떨어짐)
- 콘이 여전히 어색하면 → 콘을 아예 제거하고 트럭 지붕에 작은 아이스크림 로고/배지만 남기는 안 (더 깔끔)

승인 시 1번 안(편집 기반 리컬러)으로 진행, 1차 결과 보고 필요시 대안 전환합니다.
