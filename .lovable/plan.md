## 남은 작업: 코드 측 없음, 사용자 측 1건

### 코드 측
현재 승인된 계획의 모든 코드 변경(`session-bootstrap.functions.ts` 공용 캐시 + 개인 like 분리)이 완료되었습니다. **추가 구현할 코드는 없습니다.**

### 사용자 측 (콘솔)
- Lovable Cloud → Overview → Advanced settings에서 **DB 인스턴스 사이즈 현재값 확인**만 해두시면 됩니다.
- 리허설/행사 중 다음 지표가 나오면 그때 한 단계 상향:
  - Memory 80% 이상
  - Connections 45/60 이상

### 리허설 시 확인 포인트 (모니터링만, 코드 작업 아님)
1. 진입 스파이크 시 서버 함수 응답 시간
2. 좋아요/댓글/pin 조작 시 Realtime 이벤트 지연
3. `bootstrapSession` 캐시 히트율 (콘솔 로그에서 확인 가능하도록 원하시면 로그 추가 계획을 별도로 세울 수 있음)

### 필요 시 후속 작업(지금 하진 않음)
아래는 리허설 결과에 따라 필요해질 수도 있는 항목입니다. 지금 미리 할 필요 없음:
- 캐시 TTL 조정(5s → 3s 또는 10s)
- 캐시 히트율 로그 추가
- DB 인스턴스 상향

## 결론
지금 시점에서 예정된 코드 작업은 모두 마쳤습니다. 리허설 결과를 보고 실제 병목이 나타난 경우에만 추가 작업을 계획하는 것을 권장합니다.
